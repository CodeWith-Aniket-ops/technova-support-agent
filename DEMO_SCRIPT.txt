[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture behind the scenes.
First, when a user sends a message, it is received by the START node and immediately sent to our Security Checkpoint. (pause)
This checkpoint node screens the input for prompt injections, unauthorized software piracy keywords, and redacts PII before it reaches any LLM. (pause)
If safe, it routes the query to the Orchestrator Agent. (pause)
The Orchestrator acts as our main controller, using AgentTools to delegate tasks to specialized sub-agents: the Product Advisor for specifications and the Troubleshooting agent for technical diagnostics. (pause)
Both sub-agents connect to our local Model Context Protocol, or MCP, Server, which exposes custom tools to fetch real-time catalog specs, order tracking, and warranty terms. (pause)
Finally, for sensitive queries like warranty replacements, the Orchestrator escalates to a Manager Review node, pausing the workflow to wait for manual human approval before returning the final response. (pause)
